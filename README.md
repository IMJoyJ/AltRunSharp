# AltRunSharp

AltRunSharp 是一个基于 WPF (.NET 10.0-windows) 开发的轻量级、高性能的桌面快速启动器与脚本自动化执行平台。它通过全局热键或修饰键连击（双击/三击）唤醒，不仅支持快速匹配并启动常规本地程序，还内置了多语言脚本引擎（支持 C# 脚本、Node.js 脚本、批处理等）的后台/前台执行、顺序工作流编排、定时调度管理以及常驻后台服务的监控管理。

---

## 🚀 核心特性

### 1. 全局唤醒与多屏智能对齐
* **多种唤醒模式**：支持配置常规的全局组合快捷键（默认 `Alt+R`），还支持通过底层键盘钩子（`WH_KEYBOARD_LL`）捕获单个修饰键（如 `Ctrl` / `Shift` / `Alt`）的双击或三击动作。
* **活跃屏幕居中**：唤醒时自动获取鼠标当前所在的显示器，并在该显示器的屏幕中央弹出输入框（支持多屏与 DPI 缩放适配）。
* **无干扰隐藏**：在输入框外点击或按 `Escape` 键可自动隐藏输入框。

### 2. 智能搜索与别名匹配
* **模糊搜索**：输入字符即可在已配置的快捷命令、程序路径以及脚本命令中进行模糊搜索。
* **别名与补全**：支持为程序和脚本配置多个搜索别名（Aliases），匹配别名时支持在列表中自动补全为原始名称或以 `/` 开头的脚本命令。

### 3. 多语言脚本执行引擎
* **C# 脚本免建项目运行**：支持直接运行单个 `.cs` 脚本文件，利用新版 .NET 的 `dotnet run <script.cs>` 能力，摆脱繁琐的项目配置，实现如 C#/Python 脚本一般的即用即走。
* **Node.js 与命令行脚本**：支持直接运行 Node.js 脚本（`.js`）、Windows 批处理（`.bat`）以及独立的二进制可执行文件（`.exe`）。
* **两种运行模式**：
  * **一次性运行 (Once)**：
    * **显式模式**：弹出美观的深色独立终端输出窗口（`OutputWindow`），支持实时接收并渲染标准输出（`stdout`）和标准错误（`stderr`）流，并在退出时展示退出状态码（正常退出显示绿点，非零退出显示红点）。
    * **静默模式**：在后台默默执行，并将所有日志按日期持久化记录到 `data/logs/yyyyMMdd.log` 中。
  * **常驻服务 (Service)**：支持将脚本或程序配置为后台持续运行的“服务”（可选择开机随程序自动启动 `BootStart`），如果崩溃或意外退出将延时自动重启。
* **工作流 (Workflow)**：支持将多个已有脚本定义为步骤（Steps），组成工作流，在执行时按照定义的顺序链式依次运行。

### 4. 任务调度管理器 (ScheduleManager)
* **定时/定期触发**：支持为工作流配置自动调度。包含两种触发方式：
  * `interval`：固定时间间隔（秒数）循环执行。
  * `daily`：每日特定的时间点（"HH:mm" 列表）定时触发。
* **动态参数脚本**：在运行工作流前，支持可选地执行一个参数计算脚本（`ArgsScriptName`），并捕获其 `stdout` 输出作为后面工作流执行的参数。
* **冲突解决机制**：支持配置当上一次任务尚未执行完又被触发时的策略：
  * `parallel`：并行启动新实例。
  * `kill`：中止上一次的任务树并重新启动。
  * `skip`：跳过本次触发。

### 5. 系统集成与高级控制
* **配置热重载**：所有的配置文件持久化保存在 `data/config.json` 中。主程序具备文件变化监控功能，手动或外部修改此文件会触发热重载，无需重启即可使新热键和新配置即时生效。
* **优雅的子进程销毁**：通过 Windows `Job Object`（工作对象）管理所有由主程序启动的服务及脚本子进程。当主程序退出时，所有的子进程树都将被强制彻底关闭，根绝残留僵尸进程。
* **Windows 系统深度集成**：
  * 支持一键开启开机自启。
  * 支持为 `.exe` 文件添加 Windows 右键菜单“添加到 AltRun”，点击即可自动提取程序名和路径。
  * 具备智能 UAC 提权机制，非管理员状态下开启需修改注册表的配置时会自动向用户申请管理员权限并重载，无需事先手动以管理员运行。

## 📂 项目文件布局

```text
AltRunSharp/
├── data/                       # 数据与存储目录
│   ├── config.json             # 默认/用户持久化配置文件

│   └── logs/                   # 静默任务及服务日志存放目录
│       └── services/           # 常驻后台服务的运行日志
├── App.xaml / App.xaml.cs      # WPF 应用入口，负责托盘图标与服务初始化
├── MainWindow.xaml / .xaml.cs  # 快速启动输入框界面，焦点及搜索响应
├── SettingsWindow.xaml / .cs   # 配置中心（包含脚本编辑器、热键录制、右键菜单、自启配置）
├── OutputWindow.xaml / .xaml.cs# 实时终端输出窗口
├── HotkeyService.cs            # 全局热键与键盘底层连击钩子服务
├── RunnerService.cs            # 负责执行本地程序、一次性脚本及工作流
├── ServiceManager.cs           # 负责常驻后台服务的生命周期与异常重启管理
├── ScheduleManager.cs          # 定时调度任务轮询与冲突控制
├── ConfigService.cs            # 配置文件读写与变更自动检测
├── ScriptArgsParser.cs         # 支持 Shell 级转义和引号规则的参数解析器
├── AdminHelper.cs              # Windows 提权与 UAC 重新启动辅助类
├── AltRunSharp.csproj          # WPF 项目工程文件
└── MANUAL_TEST_GUIDE.md        # 项目的手工测试要点说明书
```

## 🛠️ 构建与运行

### 1. 开发环境要求
* **.NET SDK 10.0** 或以上版本（支持 Windows 桌面应用开发）。
* **Node.js**（若需要执行 JS 脚本，需保证 `node` 命令已添加到系统环境变量 PATH 中）。

### 2. 编译项目
在项目根目录下，使用 .NET CLI 进行编译。本项目经过严格的代码质量审计，确保满足 **0 错误、0 警告 (0w0e)**：
```powershell
dotnet build
```

### 3. 运行程序
你可以直接使用以下命令运行程序：
```powershell
dotnet run
```
或者在编译完成后，直接双击运行：
`bin\Debug\net10.0-windows\AltRunSharp.exe`。

> 💡 **提示**：程序启动后默认会**隐藏到系统托盘**，不会直接弹出窗口。您可以通过以下方式唤醒：
> * 双击或右键托盘图标。
> * 按下默认全局快捷键：`Alt+R`。
