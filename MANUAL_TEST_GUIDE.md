# AltRunSharp 手工测试指南

> 测试前请先启动程序（运行 `bin\Debug\net10.0-windows\AltRunSharp.exe` 或直接 `dotnet run`）。
> 程序启动后会**隐藏在系统托盘**，不会出现窗口。

---

## 1. 系统托盘

| 步骤 | 预期结果 |
|------|----------|
| 查看系统托盘区域（任务栏右侧） | 能看到 AltRunSharp 图标 |
| 右键托盘图标 | 弹出菜单：显示 / 设置 / 退出 |
| 点击"退出" | 程序退出，图标消失 |
| 左键单击托盘图标 | 弹出启动器输入框（居中于当前鼠标所在显示器） |

---

## 2. 全局快捷键 & 窗口唤醒

| 步骤 | 预期结果 |
|------|----------|
| 按 `Alt+R`（默认热键） | 输入框在当前鼠标所在显示器中央弹出 |
| 再次按 `Alt+R`（已显示时） | 输入框隐藏 |
| 在输入框外点击 | 输入框自动隐藏 |
| 按 `Escape` | 输入框隐藏 |
| 多显示器：鼠标移到副屏后按 `Alt+R` | 输入框在副屏中央弹出（不在主屏） |

---

## 3. 搜索 & 程序启动

> 预置了以下 LaunchItem：记事本、计算器、资源管理器、任务管理器。

| 步骤 | 预期结果 |
|------|----------|
| 按 `Alt+R` 唤出输入框，输入"记" | 列表显示"记事本" |
| 按 `↓`/`↑` | 选中项高亮移动 |
| 选中"记事本"后按 `Enter` | 记事本启动，输入框隐藏 |
| 输入"calc"（英文搜索） | 列表显示"计算器"（如路径包含 calc.exe） |
| 输入框为空时 | 显示所有配置项 |
| 输入无匹配的文字 | 列表隐藏（无结果） |

---

## 4. 脚本执行 — 显式模式（弹窗）

> 使用预置脚本 `/test_args`（JS）和 `/test_env`（C#）。
>
> ⚠️ 前提：已安装 `node` 和 `dotnet`（均在 PATH 中）。

### 4a. JS 脚本无参数
1. 按 `Alt+R`，输入 `/test_args`，按 `Enter`
2. **预期**：弹出深色终端窗口，显示"收到 0 个参数"，退出码 0（绿点）

### 4b. JS 脚本带简单参数
1. 输入 `/test_args hello world`，按 `Enter`
2. **预期**：窗口显示收到 2 个参数：`"hello"`、`"world"`

### 4c. JS 脚本带引号和空格的参数
1. 输入 `/test_args "hello world" "\"escaped\""`，按 `Enter`
2. **预期**：收到 2 个参数：`"hello world"`、`"\"escaped\""`（注意转义）

### 4d. C# 脚本
1. 输入 `/test_env`，按 `Enter`
2. **预期**：显示运行时信息、环境变量（USERNAME、COMPUTERNAME 等）

---

## 5. 脚本执行 — 静默模式（日志）

1. 输入 `/test_silent hello`，按 `Enter`（无弹窗）
2. 检查 `data\logs\` 目录下是否生成了 `yyyyMMdd.log` 文件
3. **预期**：日志文件中包含脚本输出内容和退出码

---

## 6. 设置界面

右键托盘图标 → 设置，或在菜单中选择"设置"。

### 6a. 快速启动管理
| 步骤 | 预期结果 |
|------|----------|
| 点击"快速启动"标签 | 右侧显示已配置的启动项列表 |
| 点击"+ 添加" | 列表新增一项"新程序" |
| 填写名称/路径后点"保存" | 信息保存，关闭设置后用搜索能找到新项目 |
| 选中一项，点"− 删除" | 项目从列表和配置中删除 |
| 点"浏览"按钮 | 打开文件选择器，选 .exe 后自动填入路径 |

### 6b. 脚本执行管理
| 步骤 | 预期结果 |
|------|----------|
| 点击"脚本执行"标签 | 显示已有脚本（test_args、test_env） |
| 选择脚本，修改内容后点"保存" | `data\scripts\` 下对应文件内容已更新 |
| 新建脚本，命名、选 JS/CS、填写代码后保存 | 文件生成在 `data\scripts\` |

### 6c. 配置页
| 步骤 | 预期结果 |
|------|----------|
| Ctrl 连击模式选"双击 Ctrl" | 保存后，快速双击 Ctrl 能唤出启动器 |
| 点击热键录制框 | 变为"按下目标按键..." |
| 按下 `Ctrl+Space` | 框内显示"Ctrl+Space" |
| 再次按 `Alt+R` | 原热键已失效，新热键生效 |
| 开启"开机自动启动" | 注册表 `HKCU\...\Run` 中出现 AltRunSharp 条目 |

---

## 7. 配置热重载

1. 启动器运行中，手动编辑 `data\config.json`，修改 `hotkey` 为 `"Ctrl+Q"`
2. 保存文件
3. **预期**：程序自动检测到变化，`Ctrl+Q` 立即生效，无需重启

---

## 8. 右键菜单（需管理员）

1. 在"配置"页开启"右键菜单"开关
2. 如非管理员运行：会弹出 UAC 提权请求
3. 批准后：右键任意 .exe 文件
4. **预期**：右键菜单中出现"添加到 AltRun"选项

---

## 9. 边界情况

| 场景 | 预期结果 |
|------|----------|
| 输入框中使用中文输入法 | 输入法候选框出现时，`Escape` 先关闭候选框，不关闭启动器 |
| 注册热键冲突（被其他程序占用） | 弹出非阻塞提示框，程序正常运行 |
| 系统解锁（锁屏后解锁） | 热键自动重新注册 |
| 电源恢复（睡眠后唤醒） | 热键自动重新注册 |
| 关闭设置窗口 | 启动器输入框仍可正常弹出 |

---

## 已知约束

- C# 脚本需要 `dotnet` SDK（非仅运行时）
- 右键菜单注册需要管理员权限（UAC 流程）
- 开机自启写入 `HKCU\...\Run`，无需管理员
