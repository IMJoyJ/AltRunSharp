# E2E Test Infra: AltRunSharp

## Test Philosophy
- **Opaque-box (黑盒测试)**: 测试套件不依赖任何 AltRunSharp 内部实现的模块，仅通过其公开的接口、数据文件 `data/config.json`、系统注册表、热键模拟以及 UI Automation (FlaUI) 对运行状态进行观察和操作。
- **Requirement-driven (需求驱动)**: 完全依据 `ORIGINAL_REQUEST.md` 中的所有要求进行测试设计。
- **Methodology (设计方法)**: 
  - **Category-Partition (等价类划分)**: 将输入参数、快捷键模式、搜索关键词、运行模式划分为不同等价类，确保每个等价类都有覆盖。
  - **Boundary Value Analysis (边界值分析)**: 针对空输入、超长输入、包含各种转义符/特殊字符的参数、空文件/大文件等进行边界测试。
  - **Pairwise Combinatorial Testing (两两组合测试)**: 确保 JS 脚本、C# 脚本、静默运行、显式运行、带参、不带参等维度交叉覆盖。
  - **Real-World Workload Testing (真实场景工作负载)**: 模拟用户真实的日常软件管理、开发脚本运行、环境维护等使用习惯，测试长周期和多任务场景。

---

## Feature Inventory (功能清单)
本测试套件针对以下 8 项核心功能进行验证：
1. **F1: 全局快捷键与窗口唤醒** (Alt+R / 双击 Ctrl / 三击 Ctrl)
2. **F2: 输入框失焦与 Esc 隐藏** (输入框外点击隐藏 / 按 Esc 隐藏)
3. **F3: 模糊搜索与列表导航** (模糊拼写匹配 / 上下方向键选择 / 回车运行)
4. **F4: 外部 exe 程序启动与传参** (程序路径 / 自定义命令行参数)
5. **F5: JS/C# 脚本执行引擎** (NodeJS 执行 JS / dotnet 编译运行 C#)
6. **F6: 脚本复杂参数解析与转义** (双引号 / 空格 / 转义 / 多 args)
7. **F7: 配置持久化** (读写 `data\config.json`)
8. **F8: 系统集成与权限管理** (开机自启 / 右键菜单 / UAC 提权集成)

---

## Test Architecture (测试架构)
E2E 测试套件包含两个主要部分：
1. **AltRunSharp.E2E.dll (测试库)**: 基于 NUnit 框架，使用 **FlaUI.UIA3** 库进行 Windows UI 自动化测试。测试程序在运行时会：
   - 动态修改 `data/config.json` 来预置测试用例所需的数据。
   - 使用 `Process.Start` 启动 `AltRunSharp.exe`。
   - 通过 FlaUI 获取 `MainWindow` 控件，模拟键盘输入，触发搜索和运行。
   - 使用 Win32 API `SendInput` 模拟全局热键及 Ctrl 双击/三击。
   - 监控 OutputWindow 的文本或读取 `data/logs/` 下的日志文件。
   - 检查 Windows 注册表项是否成功写入。
2. **run-e2e.ps1 (测试运行器)**: 一个 PowerShell 脚本，自动完成以下操作：
   - 编译项目及 E2E 测试项目。
   - 清理旧的测试环境（日志、临时配置）。
   - 运行 E2E 测试用例，输出详细的 TRX 报告。
   - 检查测试执行结果，根据通过率返回退出码（全部通过返回 0，否则返回非 0）。

## Test Cases (测试用例集)

### Tier 1: Feature Coverage (功能覆盖，40个用例)
*针对 8 项核心功能，每项设计 5 个测试用例，总计 40 个。*

#### F1. 全局快捷键与窗口唤醒 (F1_01 ~ F1_05)
- **T1_F1_01**: 启动程序后，按下默认热键 Alt+R，验证 `MainWindow` 被成功唤醒并置于最前。
- **T1_F1_02**: 在配置中开启“双击 Ctrl 启动”，双击 Ctrl 键，验证窗口在当前活跃屏幕居中弹出。
- **T1_F1_03**: 在配置中开启“三击 Ctrl 启动”，三击 Ctrl 键，验证窗口在当前活跃屏幕居中弹出。
- **T1_F1_04**: 托盘图标正常显示，右键托盘图标点击“显示”，验证输入框弹出。
- **T1_F1_05**: 修改全局热键为 Alt+S，按下 Alt+S 唤醒窗口，按下旧热键 Alt+R 窗口不响应。

#### F2. 输入框失焦与 Esc 隐藏 (F2_01 ~ F2_05)
- **T1_F2_01**: 唤醒输入框后，在输入框外的屏幕空白区域点击鼠标，验证窗口隐藏。
- **T1_F2_02**: 唤醒输入框后，按下键盘上的 Esc 键，验证窗口被隐藏且清空输入。
- **T1_F2_03**: 唤醒输入框后，点击任务栏其他程序窗口，验证输入框失焦隐藏。
- **T1_F2_04**: 输入框弹出，在托盘图标上点击左键，验证窗口重新隐藏（如果本来是显示的）。
- **T1_F2_05**: 弹出设置窗口 `SettingsWindow` 时，验证按 Esc 键或点击外部不会意外隐藏 `SettingsWindow`。

#### F3. 模糊搜索与列表导航 (F3_01 ~ F3_05)
- **T1_F3_01**: 输入快捷命令全名，验证匹配项高亮显示在列表第一项。
- **T1_F3_02**: 输入部分字符（如 `calc` 输入 `cc`），验证通过模糊搜索匹配到 `calc.exe`。
- **T1_F3_03**: 输入搜索词后，按向下方向键，验证选中焦点移动到第二项。
- **T1_F3_04**: 选中列表某项后，按 Enter 键，验证该程序被执行，且主输入框隐藏。
- **T1_F3_05**: 输入无法匹配任何命令的字符，验证列表显示为空，回车不发生动作。

#### F4. 外部 exe 程序启动与传参 (F4_01 ~ F4_05)
- **T1_F4_01**: 快速启动配置 `notepad.exe`，执行后验证记事本进程被启动。
- **T1_F4_02**: 配置 `cmd.exe` 带参数 `/c echo hello`，执行后验证 cmd 进程启动且完成输出。
- **T1_F4_03**: 启动一个带相对路径的 exe（如 `.\bin\helper.exe`），验证能在正确的工作目录下启动。
- **T1_F4_04**: 配置 `ping.exe` 带参数 `127.0.0.1` 启动，验证进程传参正常。
- **T1_F4_05**: 快捷启动名称包含中文字符，匹配后回车运行，验证目标程序正常启动。

#### F5. JS/C# 脚本执行引擎 (F5_01 ~ F5_05)
- **T1_F5_01**: 添加一个 JS 脚本，执行静默运行，验证调用本地 `node` 成功执行并正常退出。
- **T1_F5_02**: 添加一个 C# 脚本，执行静默运行，验证调用 `dotnet run` 成功执行并正常退出。
- **T1_F5_03**: JS 脚本执行显式运行，验证 `OutputWindow` 实时弹出并输出数据。
- **T1_F5_04**: C# 脚本执行显式运行，验证 `OutputWindow` 实时弹出并输出数据。
- **T1_F5_05**: 脚本抛出错误（例如 JS `throw new Error()`），验证在日志文件或输出窗口中记录了相应的错误栈。

#### F6. 脚本复杂参数解析与转义 (F6_01 ~ F6_05)
- **T1_F6_01**: 执行带有普通空格参数的命令（如 `/cmd a b`），验证脚本接收到 2 个 args：`a` 和 `b`。
- **T1_F6_02**: 使用双引号包裹带空格的参数（如 `/cmd "a b" c`），验证脚本接收到 2 个 args：`a b` 和 `c`。
- **T1_F6_03**: 参数中带有转义双引号（如 `/cmd "123" "\"1 2\""`），验证解析为 `123` 和 `"1 2"`。
- **T1_F6_04**: 参数包含反斜杠转义字符（如 `C:\\Windows\\Temp`），验证解析后能还原出正确的路径。
- **T1_F6_05**: 传入空参数或多个连续空格分隔的参数（如 `/cmd  a   b `），验证不会生成多余的空参数项。

#### F7. 配置持久化 (F7_01 ~ F7_05)
- **T1_F7_01**: 在设置界面添加一个快速启动项，关闭设置窗口，验证 `data\config.json` 包含该项目。
- **T1_F7_02**: 修改 `data\config.json`，增加一个新的快捷命令，重启 AltRunSharp，验证在输入框中能搜索到该命令。
- **T1_F7_03**: 在设置界面删除快捷启动项，验证 `data\config.json` 中该项已被清除。
- **T1_F7_04**: 修改全局热键设置并保存，验证 `data\config.json` 中的热键字段被更新。
- **T1_F7_05**: 损坏的 `data\config.json`（例如不合法的 JSON 格式），验证程序启动时能优雅降级（使用默认配置或自动修复）。

#### F8. 系统集成与权限管理 (F8_01 ~ F8_05)
- **T1_F8_01**: 开启“开机自启”开关（普通权限），验证程序弹出 UAC 提权请求或通过 helper 进程写注册表。
- **T1_F8_02**: 开启“开机自启”开关（管理员权限下），验证注册表 `Software\Microsoft\Windows\CurrentVersion\Run` 成功写入。
- **T1_F8_03**: 关闭“开机自启”开关，验证注册表中对应启动项被成功删除。
- **T1_F8_04**: 开启“添加到 AltRun”右键菜单开关（普通权限），验证弹出提权请求。
- **T1_F8_05**: 开启“添加到 AltRun”右键菜单开关（管理员权限），验证注册表 `Software\Classes\exefile\shell\AltRun` 成功创建。

### Tier 2: Boundary & Corner Cases (边界与极限情况，40个用例)
*针对 8 项核心功能，每项设计 5 个边界/异常测试用例，总计 40 个。*

#### F1. 全局快捷键与窗口唤醒 (F2_F1_01 ~ F2_F1_05)
- **T2_F1_01**: 系统当前有其他程序独占了 Alt+R 键，验证 AltRunSharp 启动时不会崩溃，并能在设置中更改热键或给出错误提示。
- **T2_F1_02**: 频繁、极快速地连续按下 Alt+R（如 10 次/秒），验证主输入框不会发生闪烁或多次叠加弹出，且状态保持稳定。
- **T2_F1_03**: 在鼠标移动到双显示器边界时触发双击 Ctrl，验证主输入框能正确定位在鼠标当前的显示器正中央，而不会跨屏幕显示。
- **T2_F1_04**: 将热键修改为包含 Ctrl, Alt, Shift, Win 的极复杂组合（如 Ctrl+Alt+Shift+Win+X），验证能否正常唤醒。
- **T2_F1_05**: 在休眠唤醒或锁屏解锁后，验证全局快捷键钩子依然有效。

#### F2. 输入框失焦与 Esc 隐藏 (F2_F2_01 ~ F2_F2_05)
- **T2_F2_01**: 唤醒输入框后，鼠标悬停在输入框上但点击了任务栏非活跃区，验证输入框能正常失焦隐藏。
- **T2_F2_02**: 唤醒输入框后，正处于 IME 中文输入法选词状态下，按下 Esc 键，首选取消输入法选词，再次按下 Esc 隐藏窗口。
- **T2_F2_03**: 弹出的主输入框失焦隐藏后，再次按下 Alt+R，验证其保留之前的输入文字或根据配置清空。
- **T2_F2_04**: 在输入框显示时，触发 Windows 锁屏（Win+L），解锁后验证主输入框已被安全隐藏。
- **T2_F2_05**: 点击主输入框内的滚动条或下拉菜单（非外部区域），验证输入框不会因为误判为“失焦”而隐藏。

#### F3. 模糊搜索与列表导航 (F2_F3_01 ~ F2_F3_05)
- **T2_F3_01**: 输入极其特殊的正则字符（如 `.*+?^${}()|[]\`），验证模糊搜索不发生异常报错，且能正确进行字面匹配。
- **T2_F3_02**: 搜索结果项极多（例如 > 100 个匹配项），验证界面显示流畅，滚动条正常工作，且键盘导航不会发生卡顿。
- **T2_F3_03**: 搜索词长度超过 1000 字符，验证程序截断或限制输入，且不发生内存溢出。
- **T2_F3_04**: 列表只有 1 个项时，按 Down/Up 键，验证焦点依旧保留在此项，不会发生空指针异常。
- **T2_F3_05**: 列表项包含极长的名称或详情描述，验证 UI 能正确裁剪文本并显示省略号，不破坏“顶栏-左栏-右栏”或列表布局。

#### F4. 外部 exe 程序启动与传参 (F2_F4_01 ~ F2_F4_05)
- **T2_F4_01**: 启动一个需要管理员权限运行的 exe，但以普通用户运行 AltRunSharp，验证目标程序唤起 Windows 系统的 UAC，且 AltRunSharp 进程不挂起。
- **T2_F4_02**: 执行一个路径不存在的损坏快捷启动，验证 AltRunSharp 不会崩溃，并能在界面上提示“找不到目标文件”。
- **T2_F4_03**: 启动一个启动后立即退出的 exe，验证 AltRunSharp 能正常隐藏输入框且不发生阻塞。
- **T2_F4_04**: 启动一个带有海量命令行参数（参数总长度达 8KB）的 exe，验证传参正常且进程启动成功。
- **T2_F4_05**: 目标 exe 的文件名包含空格或 Unicode 字符（如 `测试 程序.exe`），验证能否被正确包裹路径并执行。

#### F5. JS/C# 脚本执行引擎 (F2_F5_01 ~ F2_F5_05)
- **T2_F5_01**: JS/C# 脚本在后台执行死循环，验证 AltRunSharp 的 UI 绝不被卡死，且能在设置中强制结束该脚本进程。
- **T2_F5_02**: JS 脚本执行需要本地 node 环境，但在未安装 node 的干净环境下运行脚本，验证程序给出优雅提示“未检测到 NodeJS 环境”。
- **T2_F5_03**: C# 脚本在首次运行时（无 `dotnet-script` 或无 `.csproj` 编译缓存），验证编译期间输出窗口显示“正在编译/准备...”，且不卡死主窗口。
- **T2_F5_04**: 在高并发下连续快速执行 10 次同一个脚本，验证日志文件能正确多线程并发写入或按顺序安全排队写入，无文件锁定错误。
- **T2_F5_05**: 静默运行脚本生成的 log 所在磁盘空间不足，验证程序日志模块不抛出未处理异常，能优雅记录异常到事件查看器或忽略。

#### F6. 脚本复杂参数解析与转义 (F2_F6_01 ~ F2_F6_05)
- **T2_F6_01**: 传入包含多对嵌套双引号的参数（如 `/cmd "outer \"inner\" outer"`），验证正确解析出 `outer "inner" outer`。
- **T2_F6_02**: 参数包含 Windows 环境变量（如 `%TEMP%`），验证脚本接收到的是原始字符串还是解密后的环境变量（视设计而定，确保稳定性）。
- **T2_F6_03**: 参数只包含单个未闭合的双引号（如 `/cmd "unclosed`），验证参数解析器能安全处理，不发生崩溃，或将其作为整体处理。
- **T2_F6_04**: 参数包含特殊换行符或 Tab 键（例如 `\n`, `\t`），验证解析器正确转义而非中断。
- **T2_F6_05**: 参数以斜杠或连字符开头（如 `-f --config=true /p`），验证与脚本命令前缀 `/` 冲突时的解析正确性。

#### F7. 配置持久化 (F2_F7_01 ~ F2_F7_05)
- **T2_F7_01**: 在 `data\config.json` 处于只读状态下，修改设置，验证程序给出保存失败的友好弹窗，且不会强制崩溃。
- **T2_F7_02**: 配置文件 `data\config.json` 被其他进程占用锁定，验证 AltRunSharp 读取时进行重试，或者能采用备用缓存启动。
- **T2_F7_03**: 多次反复并发修改配置并保存，验证配置没有被截断或写入空数据。
- **T2_F7_04**: 配置文件大小达到数 MB（包含数千个快捷命令），验证系统启动和加载时间在 1 秒以内，性能良好。
- **T2_F7_05**: 修改配置中的保存路径到非系统盘（若支持），验证能够正确在新路径创建 `config.json` 并读取。

#### F8. 系统集成与权限管理 (F2_F8_01 ~ F2_F8_05)
- **T2_F8_01**: 在受限的 Windows 家庭版（无组策略）或被安全杀毒软件（如 360/火绒）拦截修改注册表时，验证程序能够拦截注册表写入异常，并不崩溃。
- **T2_F8_02**: 右键菜单写入了已不存在的 AltRunSharp 旧路径，验证程序更新时能自动覆盖修复该路径。
- **T2_F8_03**: 在开机自启写入时，路径包含中文字符和空格，验证注册表中已正确用双引号包裹路径（避免 Windows 自启路径解析漏洞）。
- **T2_F8_04**: 启动提权 Helper 进程时，用户在 UAC 弹窗中点击了“否”（取消），验证主程序能正确捕捉取消状态，不发生无限弹窗，开关按钮复位。
- **T2_F8_05**: 以管理员权限运行主程序时，开机自启和右键菜单开关不需要再弹出 UAC 直接修改成功。

### Tier 3: Cross-Feature Combinations (两两交互覆盖，8个用例)
*测试不同功能模块组合在一起时的交互表现。*

- **T3_01 (F1 ✕ F7)**: 修改全局快捷键 -> 保存配置 -> 重新加载配置 -> 验证新热键可用，且旧热键失效。
- **T3_02 (F3 ✕ F5)**: 模糊搜索脚本命令 -> 选中脚本并回车运行 -> 验证其成功调用 JS/CS 引擎执行。
- **T3_03 (F3 ✕ F4)**: 模糊搜索快速启动程序 -> 带自定义参数运行 -> 验证外部 exe 接收到正确参数，且主输入框隐藏。
- **T3_04 (F5 ✕ F2)**: 显式运行一个耗时 5 秒的 C# 脚本 -> 弹出 OutputWindow -> 在主输入框中按下 Esc 键 -> 验证主输入框被隐藏，但 OutputWindow 依旧显示并持续输出。
- **T3_05 (F5 ✕ F7)**: 显式执行脚本 -> 验证 OutputWindow 正常输出 -> 关闭 OutputWindow -> 在设置中将该脚本修改为“静默执行”并保存 -> 再次运行该脚本 -> 验证 OutputWindow 不再弹出，而是向 `data\logs\` 写入了当天日志。

  - 进程退出后，检查当天日志中正确记录了“Process killed by user/timeout”。
